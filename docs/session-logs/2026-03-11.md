### Decisions made
- AttachmentRepository is standalone (not extending BaseRepository) since attachments belong to compositions, not directly to ensembles — callers verify ensemble ownership via CompositionRepository first
- `findLinkedToAnyEvent` uses two queries: first selectDistinct compositionIds from eventCompositions, then fetch compositions by those IDs — avoids complex subquery
- Attachment delete extracts storage path from URL by splitting on `/storage/v1/object/public/attachments/` — Supabase-specific but matches the URL format
- Storage cleanup failure on attachment delete is caught and logged but doesn't fail the API response
- File upload uses `crypto.randomUUID()` for unique storage paths under `compositions/{ensembleId}/{compositionId}/{uuid}`
