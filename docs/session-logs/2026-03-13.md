## 15:45 ‚Äî Fix Playwright E2E Test Failures (13‚Üí0)

### What was done
- Fixed all 13 failing e2e tests (from 27 pass / 13 fail ‚Üí 33 pass / 7 skipped / 0 fail)
- **Fix 1 ‚Äî Auth race condition**: Set `fullyParallel: false` and `workers: 2` in `playwright.config.ts` to prevent parallel token refresh conflicts
- **Fix 2 ‚Äî Global signOut revoking all sessions**: Changed `supabase.auth.signOut()` to `scope: "local"` in `supabase-auth-service.ts`. The default `scope: "global"` was revoking ALL admin refresh tokens (including the storageState used by other tests) when the logout e2e test ran
- **Fix 3 ‚Äî Project-scoped test skips**: Added `test.beforeEach` skips in `navigation.spec.ts` so sidebar tests skip on mobile and bottom nav tests skip on chromium
- **Fix 4 ‚Äî Explicit storageState for admin tests**: Wrapped admin-only assertions in dedicated `test.describe` blocks with `test.use({ storageState: admin.json })` in events, compositions, and navigation specs ‚Äî ensures they pass on the mobile project too
- **Fix 5 ‚Äî Tab attribute assertion**: Changed from `data-state="active"` / `data-active=""` to `aria-selected="true"` ‚Äî radix-ui unified package doesn't set `data-active` as an HTML attribute
- **Fix 6 ‚Äî Logout test on mobile**: Skipped on mobile project since the logout button is in the sidebar (`hidden md:flex`)
- **Fix 7 ‚Äî waitForLoadState**: Added `waitForLoadState("networkidle")` to `EventsPage.goto()` and `NavigationPage.navigateTo()`

### Decisions made
- **`scope: "local"` for signOut**: Better app behavior anyway ‚Äî logging out should only clear the current session, not revoke all user sessions across devices
- **`aria-selected` over data attributes**: More reliable and semantic ‚Äî tests the actual accessibility state rather than implementation-specific HTML attributes
- **workers: 2 (not unlimited)**: Matches the 2 projects (chromium/mobile), prevents dev server overload from too many concurrent browser instances
- **beforeEach pattern for test.skip**: Playwright's `test.skip(callback)` at describe level doesn't receive `testInfo`; the `beforeEach` pattern is the correct API

### Issues encountered
- `test.skip(({ }, testInfo) => ...)` at describe level threw `TypeError: Cannot read properties of undefined (reading 'project')` ‚Äî Playwright's describe-level skip callback doesn't receive testInfo
- First attempt with `data-active=""` attribute assertion failed ‚Äî radix-ui sets `[active]` ARIA state but not a `data-active` HTML attribute
- With `workers: undefined` (6 workers), dev server timed out on most requests ‚Äî too many concurrent browser instances overwhelmed localhost:3000
- With `workers: 1`, discovered the global signOut issue: auth logout test revoked ALL admin sessions, causing every subsequent test using admin.json to land on the login page
